For most projects, the best way to start is the combo of `@useprint/components` for authoring and `@useprint/cli` for local preview.

## Install the starter combo

```bash
npm install @useprint/components
npm install -D @useprint/cli
```

This gives you:

- React document primitives such as `Document`, `Head`, `Body`, and `Page`
- Access to rendering and Tailwind helpers through the component package
- A local preview and export workflow through the `useprint` CLI

## Add a documents directory

By default, the CLI looks for documents in `./documents`.

```text
documents/
  invoice.tsx
  static/
```

- Put your document components directly in `documents/`
- Put images, fonts, and other static assets in `documents/static/`

## When to install packages directly

Install dedicated packages when you want tighter control over the stack:

```bash
npm install @useprint/render @useprint/tailwind
```

- Use `@useprint/render` when you only need the React-to-HTML layer
- Use `@useprint/tailwind` when you want Tailwind classes applied during rendering
