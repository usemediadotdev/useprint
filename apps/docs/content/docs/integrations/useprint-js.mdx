---
title: useprint-js
description: The recommended hosted path for turning React or HTML into PDFs.
---

`useprint-js` is the recommended way to move from React documents to generated PDFs.

It accepts either:

- a React node, which it renders to HTML before upload
- a raw HTML string, if you already control the rendering layer yourself

## Install

```bash
npm install useprint-js @useprint/components
```

## Print a React document

```tsx
import { Body, Document, Head, Page } from '@useprint/components';
import { usePrint } from 'useprint-js';

function Receipt() {
  return (
    <Document pageSize="A4">
      <Head />
      <Body>
        <Page style={{ padding: 32 }}>
          <h1>Receipt</h1>
          <p>Rendered from React, delivered as a PDF.</p>
        </Page>
      </Body>
    </Document>
  );
}

const client = usePrint({
  apiKey: process.env.USEPRINT_API_KEY,
});

const pdf = await client.print(<Receipt />, {
  pdfOptions: {
    format: 'A4',
    orientation: 'portrait',
  },
});
```

## Print raw HTML

```ts
import { usePrint } from 'useprint-js';

const client = usePrint({
  apiKey: process.env.USEPRINT_API_KEY,
});

const pdf = await client.print('<html><body><h1>Hello</h1></body></html>', {
  pdfOptions: {
    format: 'A4',
  },
});
```

## How it fits with Useprint

When you pass React into `useprint-js`, the React tree is first rendered to HTML through `@useprint/render`. That makes `useprint-js` the easiest production path when you want:

- React-based authoring
- HTML output you do not have to manage manually
- a hosted PDF step instead of your own browser automation stack

## When to choose something else

If you want full control over the browser, queueing, or infrastructure, render the HTML yourself and drive Chromium directly.
